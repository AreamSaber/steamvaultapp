# Steam Vault Android Handoff

## 1. 当前状态

- 技术路线已确定为原生 Android + Kotlin + Jetpack Compose。
- 当前仓库已经不是模板工程，主密码、加密 Vault、导入、验证码、本地备份恢复、WebDAV 手动云备份都已可用。
- 当前主线阶段是“P2 收口 + P3 前置准备”，不是早期原型阶段。

## 5. 当前边界

- 目前云备份已经支持进程内自动上传排队 / 重试，以及最近 5 份远端历史版本。
- 目前手动恢复会主动清空待上传的自动备份队列；如果有自动上传正在进行，会等待它结束后再恢复。
- 目前恢复完成后，自动备份状态会暂停到下一次本地变更，而不是立刻重新上传刚恢复的内容。
- 目前已经有 WorkManager 后台补偿同步，但还没有做完整的真机 WebDAV 手动闭环联调验收。
- 主密码修改代码链路已经完成，但本地恢复 / 云恢复回归验收还没有补完。
- 目前 Steam 会话已经支持本地手动录入 / 更新，以及通过内置浏览器自动抓取 `steamcommunity.com` 登录会话；但还没有真实 Steam 登录 DTO / API、确认列表与批准 / 拒绝主链路。
- 主要中文文案与用户可见错误提示已完成一轮 `strings.xml` 收口。

## 6. 当前推荐顺序

这里的顺序需要与 `docs/ANDROID_EXECUTION_CHECKLIST_P2_P4.md` 保持一致：

1. 先做 P2 验收收尾：真机 WebDAV 上传 / 下载 / 恢复闭环验证，以及主密码修改后的本地恢复 / 云恢复回归测试。
2. 再推进 Steam 登录会话、确认列表、批准 / 拒绝等 P3 主链路。
3. 最后再做 P4 的长期安全增强。
