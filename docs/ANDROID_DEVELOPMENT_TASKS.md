# Steam Vault App 原生 Android 开发任务清单

## 1. 文档定位

- 本文档是长期里程碑总览，负责描述 P0 到 P4 的整体路线。
- 当前阶段的具体执行顺序、细化任务和测试要求，统一维护在 `docs/ANDROID_EXECUTION_CHECKLIST_P2_P4.md`。
- `docs/HANDOFF_2026-04-07.md` 只记录当前状态、最近修复和最近推荐顺序，不再额外衍生新的规划文档。

## 2. 当前阶段判断

- 当前工程已经明显超过 P1 离线原型阶段。
- 当前主线处于“P2 收口 + P3 前置准备”阶段。
- 现有代码已经覆盖：主密码、加密 Vault、本地导入、验证码、本地加密备份恢复、WebDAV 云备份 2.0 首版（手动上传恢复、自动触发、失败重试、多版本保留、恢复交互规则）、Steam 时间同步、Steam 会话存储底座。

## 3. 目标里程碑

### P0 基础架构

目标：建立可持续开发的 Android 工程骨架与分层边界。

- [ ] 稳定导航、主题、页面状态模型
- [ ] 稳定 Repository 与安全层边界
- [ ] 稳定异常提示与基础测试脚手架

### P1 离线原型

目标：完成单设备、离线可用的 Steam Guard 核心闭环。

- [ ] 主密码创建、解锁、锁定
- [ ] 本地加密存储与启动后恢复
- [ ] 令牌导入、令牌列表、验证码生成
- [ ] 基础异常处理与安全保护

### P2 MVP

目标：完成本地加密备份恢复与跨设备密文恢复能力。

- [ ] 多账户管理体验收口
- [ ] 本地加密备份导出 / 恢复收口
- [ ] WebDAV 云备份从手动 1.0 走向可长期使用的 2.0
- [ ] 恢复、锁定、错误提示与状态展示一致化

### P3 功能增强

目标：把当前“令牌器 + 备份器”升级为更完整的 Steam 管理工具。

- [ ] Steam 登录会话闭环
- [ ] 待确认列表
- [ ] 批准 / 拒绝确认
- [ ] 更多操作状态反馈与会话过期处理

### P4 安全增强

目标：在主链路稳定后，补长期安全能力与质量项。

- [ ] 主密码修改与 Vault key 重包装
- [ ] 密钥轮换与长期迁移策略
- [ ] 自动备份策略与系统备份风险审查
- [ ] 更完整的安全回归测试矩阵

## 4. 当前建议开发顺序

说明：本节只给出高层顺序，具体 checklist 以 `docs/ANDROID_EXECUTION_CHECKLIST_P2_P4.md` 为准。

1. 先做资源化收口，把主要中文文案、错误提示、状态文本抽到 `res/values/strings.xml`。
2. 再做 WebDAV 云备份 2.0 收口：系统级后台同步方案、真机联调与闭环验证。
3. 补主密码修改后的本地备份 / 云备份 / 恢复链路回归验收。
4. 然后推进 Steam 登录会话、确认列表和批准 / 拒绝等 P3 主链路。
5. 最后做 P4 的长期安全增强与更高强度密码学演进。

## 5. 当前仍需确认的决策

- [ ] 云端长期路线是否继续只保留 WebDAV，还是后续补更完整同步服务
- [ ] 云备份历史版本默认保留数量与清理策略
- [x] 主密码修改后，默认清除生物识别快速解锁并提示重新启用
- [ ] Steam 登录 / 确认系统的首版范围是否只覆盖查看与批准 / 拒绝，不扩展到更大 Steam 功能

## 6. 当前明确不做的范围

- [ ] 交易确认以外的更大 Steam 自动化能力
- [ ] 推送审批系统
- [ ] 桌面端同步客户端
- [ ] 一次性引入过度复杂的多层嵌套加密方案
- [ ] 在没有回归测试之前大幅替换现有 Vault 主加密格式
