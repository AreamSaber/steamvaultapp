```text
你是“移动端产品设计负责人（UX/UI）+ Android Jetpack Compose / Material 3 UI 架构师 + 中文 UX Writer + 视觉设计总监”。
你将对一个名为 SteamVault（中文：Steam 令牌库）的 Android 应用进行“全量前端重写”：包含信息架构、页面布局、组件系统、交互细节、状态管理呈现、视觉规范、以及所有用户可见文案（标题/说明/按钮/空状态/错误/成功/提示/高级说明）。

【你必须先做的第一步：扫描项目】

【产品定位（你要把它做成什么样）】
把 SteamVault 做成“像金融类安全应用一样可信赖”的体验：冷静、克制、干净、信息层级清晰、极少的噪音。
用户打开就知道：
1) 这是什么：Steam 验证码/交易确认的安全管家
2) 为什么可信：本地加密、敏感信息默认隐藏、可选生物识别、可备份与恢复
3) 如何完成目标：添加账号 → 查看验证码 → 保存会话 → 处理确认/扫码登录

【审美与布局方向（必须明显变好看）】
你要显著改善“当前不美观”的原因：信息堆叠、控件缺少层级、对齐与间距不足、卡片/输入框太“硬”、缺少主次 CTA、缺少漂亮的空状态与引导。

必须落实的设计系统（Material 3 风格但更高级）：
- 使用更“干净”的背景：少用大面积渐变；用轻微噪点/微渐变或纯色 surface；暗色模式下也要有层级（surface/surfaceVariant）。
- 统一 8dp 网格：页面边距 20~24dp；Section card 内边距 16~20dp；控件间距 8/12/16/24 递进。
- Typography：
  - 页面标题（headlineSmall/Medium）更大更清晰
  - 说明文案（bodyMedium）尽量 1~2 行，避免长段“工程说明”
  - 关键数字（验证码/倒计时）用等宽字体 + 视觉强调
- CTA 规则：
  - 每页只有一个“主按钮”（Primary），其余是次按钮/文本按钮
  - 主按钮固定在底部或页面末尾，避免用户找不到下一步
- 状态系统：
  - 每个页面必须有：加载态、空态、错误态、成功反馈（SnackBar/Inline banner）
  - 空态必须告诉用户“下一步做什么”，不要只说“没有数据”
- 可访问性：
  - 所有按钮/图标必须有语义描述
  - 可点击区域 >= 48dp
  - 对比度与动态字体缩放可读

【信息架构（让用户不迷路）】
底部导航建议保持 3 个入口，但重新命名/呈现更产品化：
1) 令牌（Tokens）
2) 添加（Add）
3) 设置（Settings）

把“复杂能力”收敛成清晰路径：
- 最常用路径：添加账号 → 看到验证码 → 复制验证码
- 进阶路径：为账号保存 Steam 会话 → 打开确认列表 → 批准/拒绝
- 特殊路径：扫码登录批准（独立入口，但文案要解释“它不是确认列表”）
- 安全路径：备份/恢复/云备份（像密码管理器一样做）

【文案体系（必须从工程化变成用户化）】
语气：简短、明确、可信、不过度恐吓；给“可执行建议”。
必须隐藏工程词汇（默认不出现，放到“高级信息”里）：
- Vault key / 草稿 / 上下文 / 材料 / binding prep / token_gid / identity_secret / shared_secret / device_id / OAuth token / cookie
这些词如果必须展示：统一放在“高级信息（折叠）”区域，用易懂解释 + 一键复制。

术语统一建议：
- “令牌库”= 存放 Steam 验证码的安全空间
- “主密码”= 解锁本地加密数据
- “Steam 会话”= 用来加载确认列表/提交批准的登录态（可过期）
- “确认”= 交易/市场/登录批准项（confirmations）
- “扫码登录”= 批准电脑端 Steam 登录请求（不是 confirmations）

错误提示模板（必须可行动）：
- 发生了什么（1 句） + 为什么（可选） + 你现在可以做什么（按钮/下一步）
例如：
  - “会话已过期。请重新登录 Steam 以继续查看确认列表。”[重新登录]
  - “未同步 Steam 时间。先同步一次，验证码与确认签名会更准确。”[一键同步]

【实现约束（重要）】
- 不改业务逻辑（仓储层、网络层、加密算法、数据结构），只改 UI 与 strings。
- 所有文案必须进入 string resources，禁止硬编码。
- 允许重命名/合并/清理 string key，但必须一次性做干净：避免 values 与 values-zh-rCN 冲突；避免重复语义多个 key。

【你必须交付的成果】

2) 全局组件库（SectionCard、PrimaryButton、EmptyState、InlineBanner、SensitiveFieldRow、TokenCard 等）
3) 逐 Screen 的 UI 重写（Compose 代码按文件输出 patch）
4) 逐 Screen 的文案重写（strings 资源按文件输出 patch）
5) 验收清单：确保“更美观、更直观、更像真实产品”
```

## 三、逐页“单独文案”建议（中文为主，落地到 strings）

> 目标：让每一页像真实用户产品。每页都给出：**页面标题、短说明、核心区块标题/说明、主按钮、次按钮、空状态、错误/成功提示**。  
> 你可以直接把这些文案映射进 strings key（建议按 screen 分组命名）。

### 1) 欢迎页（WelcomeScreen）
**页面标题**：欢迎使用 SteamVault  
**一句话说明**：把 Steam 验证码与确认操作安全地留在你的手机里。  
**要点列表**：
- 本地加密保存，默认不上传  
- 敏感信息默认隐藏，可用生物识别快速解锁  
- 支持备份与恢复，换机不慌  
**主按钮**：开始使用  
**次按钮**：从备份恢复  
**底部小字**：SteamVault 与 Steam 官方无关联。请妥善保管你的备份与主密码。

### 2) 创建主密码（CreatePasswordScreen）
**页面标题**：设置主密码  
**一句话说明**：主密码用于解锁本地加密数据。我们无法替你找回。  
**输入框标签**：主密码 / 确认主密码  
**规则标题**：建议强度  
**规则项（更用户化）**：
- 至少 10 位  
- 建议包含字母与数字  
- 两次输入一致  
**主按钮**：完成设置  
**次按钮（可选）**：了解“为什么需要主密码”  
**错误**：
- 密码太短：主密码至少需要 10 位  
- 不一致：两次输入不一致，请再检查一次  
- 保存失败：保存失败，请重试（如仍失败，请重启应用）

### 3) 解锁（UnlockScreen）
**页面标题**：解锁令牌库  
**一句话说明**：输入主密码以继续。  
**输入框标签**：主密码  
**主按钮**：解锁  
**次按钮**：从备份恢复  
**可选入口**：使用生物识别解锁  
**错误**：
- 密码不正确：主密码不正确，请重试  
- 暂时不可用：当前无法解锁，请稍后再试

### 4) 修改主密码（ChangePasswordScreen）
**页面标题**：修改主密码  
**一句话说明**：修改后，你需要用新主密码解锁。  
**输入框**：新主密码 / 确认新主密码  
**主按钮**：保存  
**成功**：主密码已更新  
**提示**：如果你启用了生物识别快速解锁，修改后需要重新启用一次。

### 5) 令牌列表（TokenListScreen）
**页面标题**：令牌  
**一句话说明**：点击卡片查看详情，长按或菜单可复制验证码。  
**主按钮（空态）**：添加 Steam 账号  
**次按钮（空态）**：我有密钥，直接导入  
**空态标题**：还没有令牌  
**空态说明**：先添加一个 Steam 账号，验证码会在这里自动生成。  
**卡片字段建议（文案）**：
- 按钮：复制验证码  
- 小字：本轮剩余 %d 秒  
**错误**：加载失败，请下拉重试

### 6) 令牌详情（TokenDetailScreen）
**页面标题**：令牌详情  
**主区标题**：当前验证码  
**主区说明**：每 30 秒刷新一次。  
**主按钮**：复制验证码  
**次按钮**：显示/隐藏敏感信息  
**功能入口**：
- Steam 会话（用于确认与扫码登录）→ 打开 Steam 会话  
- Steam 确认（交易/市场/登录）→ 打开确认列表  
**高级信息（折叠标题）**：高级信息  
**高级信息说明**：这些内容用于排错或迁移，请谨慎分享。  
**错误**：
- 未找到令牌：该令牌不存在或已被删除  
- 密钥无效：密钥格式不正确，请重新导入

### 7) 添加账户入口（ImportTokenScreen）
**页面标题**：添加 Steam 账号  
**分区 A（推荐）标题**：登录添加（推荐）  
**分区 A 说明**：在应用内完成登录与验证，自动把验证码保存到令牌库。  
**分区 A 按钮**：开始登录  
**分区 B（兼容导入）标题**：已有密钥？直接导入  
**分区 B 说明**：支持 maFile / JSON / otpauth URI / shared_secret。  
**分区 B 按钮**：导入密钥  
**输入区标题**：导入内容  
**输入区占位**：粘贴 maFile、JSON、otpauth URI，或留空改为手动填写  
**手动区**：账号名称 / shared_secret  
**主按钮**：保存到令牌库  
**错误**：导入失败，请检查内容格式

### 8) 协议登录（SteamProtocolLoginScreen）
**页面标题**：登录 Steam  
**一句话说明**：仅用于连接 Steam 官方服务以添加账号。  
**输入框**：账号名 / 密码  
**主按钮**：继续  
**加载文案**：正在登录…  
**验证挑战（统一标题）**：需要验证  
**挑战说明（按类型）**：
- 邮箱验证码：请输入发送到 %s 的验证码  
- Steam Guard：请输入当前 5 位 Steam Guard 验证码  
- 设备确认：请在 Steam App 中批准本次登录后再继续  
**错误**：登录失败，请重试（如频繁失败，请稍后再试）

### 9) 浏览器登录捕获（SteamAddAuthenticatorScreen）
**页面标题**：完成登录并继续  
**一句话说明**：请在内置页面完成 Steam 登录，完成后应用会自动继续。  
**状态文案（短句）**：
- 等待输入账号密码  
- 等待邮箱验证码  
- 等待 Steam Guard 验证码  
- 需要额外验证（如验证码/人机验证）  
- 登录完成，准备继续  
**主按钮**：重新加载登录页  
**成功**：已获取登录信息，可以继续下一步  
**次按钮**：保存为草稿（可选）/ 清除草稿（可选）

### 10) 绑定与激活（SteamAuthenticatorBindingScreen）
**页面标题**：添加手机验证器  
**一句话说明**：接下来会向 Steam 请求验证码密钥，并把结果安全保存到本地。  
**步骤（Stepper 标题）**：
1. 检查登录状态  
2. 请求验证码密钥  
3. 输入激活码  
4. 保存到令牌库  
**主按钮（阶段性）**：
- 请求验证码密钥  
- 完成并保存  
**激活码输入说明**：请输入 Steam 发送的激活码（短信/邮件）  
**成功**：已添加成功  
**常见失败（带可行动作）**：
- 会话过期：登录已过期，请重新登录后再试 → [重新登录]
- 需要手机号：该账号需要先绑定手机号 → [去绑定手机号]
- 已有验证器：该账号已绑定验证器，请改用“兼容导入” → [去导入]
- 被限流：请求过于频繁，请稍后再试 → [我知道了]

### 11) Steam 会话（SteamSessionScreen）
**页面标题**：Steam 会话  
**一句话说明**：用于加载确认列表与提交批准操作。会话可能会过期。  
**分区 A 标题**：当前状态  
**分区 A 文案**：
- 未保存：还没有保存会话  
- 已保存：会话已保存（建议定期验证）  
**主按钮**：登录并保存会话  
**次按钮**：验证会话 / 清除会话  
**手动输入区标题**：手动补充（高级）  
**手动输入说明**：仅在自动保存失败时使用。  
**错误**：保存失败，请重试；如仍失败，请改用手动补充

### 12) Steam 确认列表（SteamConfirmationsScreen）
**页面标题**：Steam 确认  
**一句话说明**：这里显示待批准的交易、市场上架与登录确认。  
**主按钮**：刷新  
**空态标题**：暂无待确认项目  
**空态说明**：如果你刚发起交易或上架，请稍等片刻后刷新。  
**列表项按钮**：批准 / 拒绝  
**成功**：已批准 / 已拒绝  
**错误**：
- 会话缺失：需要先保存 Steam 会话 → [去保存会话]
- 时间未同步：建议先同步 Steam 时间 → [一键同步]

### 13) 扫码登录批准（SteamQrLoginScreen）
**页面标题**：扫码登录  
**一句话说明**：用于批准电脑端 Steam 登录请求（与“确认列表”不同）。  
**步骤**：
1) 粘贴二维码链接  
2) 选择要使用的账号  
3) 查看请求信息并决定是否批准  
**输入框标签**：二维码链接  
**占位**：粘贴 https://s.team/q/...  
**主按钮**：读取请求  
**批准按钮**：批准登录  
**拒绝按钮**：拒绝登录  
**成功**：已批准该登录 / 已拒绝该登录  
**错误**：读取失败，请检查链接或稍后重试

### 14) 设置（SettingsScreen）
**页面标题**：设置  
**分区标题**：
- 安全  
- 时间同步  
- 备份与恢复  
- 云备份  
**安全开关文案**：
- 保护敏感页面：开启后会阻止截图与后台预览  
- 生物识别快速解锁：解锁后可用指纹/面容快速进入  
**主操作**：
- 修改主密码  
- 立即锁定  
- 同步 Steam 时间（含最近同步时间与偏移）  
- 导出备份 / 恢复备份  
- 查看云备份状态 / 编辑云备份配置

### 15) 导出备份（BackupExportScreen）
**页面标题**：导出备份  
**一句话说明**：备份文件已加密，可用于换机或紧急恢复。  
**风险提示（更克制）**：请像保管密码一样保管备份文件。  
**主按钮**：导出加密备份  
**成功**：备份已保存到 %s  
**错误**：导出失败，请重试

### 16) 恢复备份（BackupRestoreScreen）
**页面标题**：恢复备份  
**一句话说明**：恢复会覆盖本机现有数据，请确认选择正确的备份文件。  
**主按钮**：选择备份并恢复  
**成功**：恢复完成，请使用备份对应的主密码重新解锁  
**错误**：恢复失败，请检查文件是否完整

### 17) 云备份状态（CloudBackupStatusScreen）
**页面标题**：云备份  
**一句话说明**：云端仅保存加密备份文件。  
**状态区**：
- 未配置：还没有连接云备份 → [去配置]
- 已配置：显示服务地址、账号、远端路径  
**主按钮**：上传最新备份  
**次按钮**：从云端恢复  
**列表标题**：可恢复版本  
**空态**：云端还没有备份版本

### 18) 云备份配置（CloudBackupConfigScreen）
**页面标题**：配置云备份  
**一句话说明**：连接 WebDAV（如坚果云）用于存放加密备份。  
**输入框**：WebDAV 地址 / 用户名 / 应用专用密码 / 远端路径  
**提示**：建议使用应用专用密码，而不是网页登录密码  
**主按钮**：保存配置  
**成功**：配置已保存  
**错误**：保存失败，请检查地址与账号信息

## 四、（可选）把“好看”的要求落地成验收标准

1) 每页首屏：标题 + 一句话说明 + 主要动作一眼可见（无需滚动才能找到 CTA）  
2) 每个复杂流程页（登录/绑定/会话/扫码）：都有 stepper 或“当前进度 + 下一步”提示  
3) 空态不再“空”：必须给出下一步按钮  
4) 错误提示不再“泛”：必须带行动按钮或明确下一步  
5) 敏感信息默认折叠，且复制动作明显  
6) 列表页（令牌/确认）：卡片排版统一、间距一致、信息层级清晰
